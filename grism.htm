The Smartest L2-L7 Packet Processing System

Keep your network visible, Take actions!

Mirroring traffic on IP network is more important than ever. The traffic, a huge packet stream, is just the data source to be consumed by many packet-based analysis applications such as trouble shooting system, intrusion detection, virus scan, traffic analysis, security event alerts and even behavior discovery. It is definitely an important agenda of today: to make sure the traffic is visible to those applications and those application process the packet stream efficiently.

In traditional ways, those application systems access the source from the span ports of network devices or TAPs. However, each application has its preferred packets for its unique interest so that processing the whole packet stream without selection for each is a waste. In fact, the preferred packets may even appear in more than one ports of different network device. It further implies the application system has to equip many physical ports to connect to span ports or TAPs although only a few of packets in the source are needed. This encourages the approach as below:

To aggregate the mirrored traffic from several span ports or TAPs on the production IP network and then distribute/replicate to multiple applications according to different interest.

The performance of those application systems can only decay when IP traffic and network complexity grow exponentially. We had better to adopt this approach before the packet flood burst the application systems. Otherwise, the traffic will be too big to be visible for each application very soon.

Answer for today and future: GRISM

GRISM is powered by PacketX DPI engine to tackle the challenges and realized the approach mentioned above.

Accurate Delivery

It aggregates several inputs and accurately delivers the right packets to the right application not only by L3/L4 header but also the sophisticated patterns. To improve the delivery accuracy, GRISM has the hardware-accelerated regular expression engine for the sophisticated pattern matching such as: filter the sessions for the specific HTTP URL, filter the SIP messages for the given SIP URI or filter all the FTP protocol packets (not by FTP port number).

The filtered packet stream can be distributed to more than one ports. GRISM can adopt two balance strategies:
(1) the packets in the same session are delivered to the same destination;
(2) the packets for the same subscribers are delivered to the same destination. The second one is suitable for ISP or 3G/LTE operators for ID-aware visibility.

Easy Configuration

GRISM also provides the web-based GUI for easy configuration and management. Users can quickly build the complicated filter rules which is a union or intersection of several atomic filters. Moreover, users can create a graph to describe the relation between input ports/filter rules/output ports for a complete network visibility architecture.

Robustness and Scalability

GRISM T1 is a 1U appliance with the carrier-grade robustness. Several GRISM T1s can form a packet processing pool with linear scalability while single GRISM T1 may fit small/middle-scale network. It is definitely the best choice for a progressively developing network to justify the investments on many existed and future applications.
