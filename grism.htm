The Pivot to Boost Your Analysis Power

Mirroring IP network traffic is more important than ever. Network traffic, huge packet stream, is raw source to be consumed by varied analysis devices for security, network performance, application performance, audit and even behavior discovery… Those results of analysis are critical for the network owner to form the business decisions. It then becomes a key agenda today: manage traffic to be delivered /visible to analysis systems in an efficient way. Traditionally, those analysis systems access the source from the span ports of network devices or TAPs. However, each system has its preferred packets, comes from many, for its interest: to process every packet without selection is a waste. That is, both CPU efficiency and the port usage of analysis system are low. This encourages the approach as below: 

To aggregate the mirrored traffic from several span ports or TAPs on the production IP network and then distribute/replicate to multiple analysis system according to different interest.

Without changing, the overall performance can only decay when network traffic and network complexity grow exponentially.  The network owner had better to adopt this approach before the packet flood burst the analysis systems and even ruin the new business opportunities. Otherwise, the traffic will be too big to be visible very soon.  

Answer for today and future: GRISM

GRISM, the intelligent network packet broker, is powered by PacketX’s DPI engine and U-Defined Networking, UDN, framework to tackle the challenges and realized the approach mentioned above.

Filter and Accurate Delivery

It aggregates several inputs and accurately delivers the right packets to the right system not only by L2-L4 header(VLAN ID, Tag Priority, Ethertype,IP address, IP range,TCP/UDP port…) but also the sophisticated patterns. For delivery accuracy, GRISM has the hardware-accelerated DPI engine for the sophisticated pattern matching such as: filter the sessions by the specific HTTP URL, filter the SIP messages by the given SIP URI or filter all the RTP protocol packets. The pattern can be an unbounded ASII string, HEX or the regular expression.

The filtering capability of GRISM can be session-aware. The whole session would be sent to the same destination if any packet in that session is selected. It is helpful for the analysis systems since their processing functions rely on session integrity.

Moreover, GRISM’s Equal-Distribution mechanism is different from the ACL-based distribution which is implemented in many network packet brokers or switch. GRISM can satisfy each analysis system simultaneously while other network packet brokers deliver each packet to only one system that against demand intersection.

5-tuple Load Balance

The filtered packet stream can be distributed to more than one port. GRISM can adopt the 5-tuple session-based balancing policy to guarantee “the packets in the same session are delivered to the same destination” which is critical to the analysis systems. High availability is also achieved by redistributing the stream into live links or triggering stream fail-over to the stand-by links.

Visualize the Visibility Architecture

GRISM UDN framework has the web-based GUI, GRISM web console, for configuration and management. Users can easily design the complex filter which is a union (or) or intersection (AND) of several atomic filtering rules. Moreover, the user-specific visibility architecture can be designed by manipulating the graph which represents the relation between ports/filters and configuration. All the filtering and load balance features addressed before can be achieved on this GUI.

Robustness and Scalability

GRISM-T2G8 is a carrier grade 1U appliance with 10G and 1G port so that suitable for SMB with 1G/10G co-existed network. GRISM-T16 is the scalable ATCA-based carrier-grade system for large enterprises or telco for building up the visibility platform to boot their analysis power.
